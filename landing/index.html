Core pipeline

A living model of what matters to you.

PersonaLayer turns behavior into evidence-backed persona signals, then serves only the personal context each app is allowed to receive.

Collect signals

Browser, LLM chats, MCP traffic, coding tools, CLI agents, feeds, GitHub, productivity apps, and local models.

Extract structure

Convert activity into interests, work domains, skills, tools, decisions, privacy rules, preferences, and goals.

Build the persona

Merge cross-source signals with recency, confidence, evidence, feedback, importance, and temporal scope.

Serve scoped context

AI apps declare a use case, receive allowed fields through MCP or HTTP, and leave a complete access trail.

Local-first control

Raw activity is not the product. Permissioned context is.

Each persona signal should carry source, timestamp, confidence, importance, privacy level, evidence references, and user feedback state.

Deny by default

Private and sensitive data never becomes casual context.

Raw browsing history, private content, tokens, health, finance, credentials, and sensitive identity data are blocked unless the user explicitly changes the rule.

Contracts per app

Every AI tool must declare why it needs context.

A coding agent gets stack, repo, style, and current task. A writing app gets voice and goals. Neither gets unrelated private fields.

User correction

The persona has to be editable, not mystical.

Users can confirm, reject, hide, boost, correct, set temporal scope, and lower or raise privacy on any inferred signal.
